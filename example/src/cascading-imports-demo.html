Cascading Imports Demonstration

This page demonstrates the proper use of Unify's Cascading Imports system with data-import, slot, and data-target attributes.

How It Works

1. Layout Import

This page imports the layout using data-import="/_includes/layout.html" on the root <html> element.

2. Slot Targeting

The <title> element uses data-target="title" to replace the title slot in the layout.

3. Default Slot

All content not specifically targeted goes into the unnamed/default slot in the layout's <main> element.

4. Head Merging

The <head> content from this page is intelligently merged with the layout's head, with page content taking precedence.

Key Features

  • Clean HTML Syntax: No special DSL, just standard HTML with data attributes
  • Predictable Cascade: Layout → Components → Page (like CSS layers)
  • Smart Deduplication: Automatic handling of duplicate meta tags and assets
  • Fallback Content: Slots can have default content when not overridden
  • Nested Composition: Layouts can import other layouts, components can have slots

Example Code

<!-- Layout file with slots -->
<html>
  <head>
    <slot name="title"><title>Default Title</title></slot>
  </head>
  <body>
    <header>
      <slot name="header">Default Header</slot>
    </header>
    <main>
      <slot><!-- Default content slot --></slot>
    </main>
  </body>
</html>

<!-- Page using the layout -->
<html data-import="/layouts/base.html">
  <head>
    <title data-target="title">My Page</title>
  </head>
  <body>
    <div data-target="header">Custom Header</div>
    <!-- This content goes to default slot -->
    <p>Page content here...</p>
  </body>
</html>
